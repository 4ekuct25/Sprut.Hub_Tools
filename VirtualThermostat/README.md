# 🌡️ Виртуальный термостат
Логический сценарий

> **Создание полнофункционального термостата из датчика температуры и реле**

Логический сценарий для Sprut.Hub, который позволяет создать полнофункциональный термостат, имея только датчик температуры и реле (выключатель или розетку). Сценарий автоматически управляет нагревом и охлаждением в зависимости от текущей и целевой температуры, а также автоматически регулирует скорость вентилятора (если поддерживается).

---

## 💡 Зачем это нужно?

**Проблема:** Нужен термостат для автоматического поддержания температуры в помещении, но готового устройства нет или оно дорогое. Есть только датчик температуры и реле для управления обогревателем/кондиционером.

**Решение:** Сценарий превращает виртуальное устройство "Термостат" в полнофункциональный термостат. Он автоматически получает данные с датчика температуры, отслеживает состояние датчика и управляет реле нагрева/охлаждения в зависимости от текущей и целевой температуры. Поддерживает автоматическое управление скоростью вентилятора на основе разницы температур.

---

## 📋 Пошаговая инструкция

### Шаг 1: Подготовка устройств

Убедитесь, что у вас есть:
- **Датчик температуры** (TemperatureSensor) или другой Термостат (Thermostat) с характеристикой CurrentTemperature
- **Реле нагрева** (Switch или Outlet) — для управления обогревателем
- **Реле охлаждения** (Switch или Outlet) — для управления кондиционером (опционально, если нужен только один режим)

### Шаг 2: Создание виртуального устройства

1. Создайте виртуальное устройство типа **"Термостат"** (Thermostat)

### Шаг 3: Настройка режимов термостата

1. Откройте настройки созданного виртуального устройства
2. Перейдите в **"Виртуальное устройство"** → **"Целевой режим термостата"**
3. Отключите ненужные режимы:
   - **Эко** — отключите (если не нужен)
   - **Осушитель** — отключите
   - **Вентилятор** — отключите
   - Если нужен только нагрев — отключите "Охлаждение"
   - Если нужен только охлаждение — отключите "Нагрев"
   - Режим **"Автоматически"** должен быть включён только если нужны оба режима ("Нагрев" и "Охлаждение"). В ином случае его тоже отключите.

### Шаг 4: Применение базовой логики

1. В настройках виртуального устройства перейдите в **"Виртуальное устройство"** → **"Логика"**
2. Выберите сценарий **"Обычный термостат"**
3. Активируйте переключатель **"Активно"**

### Шаг 5: Применение логического сценария

1. Скачайте файл `VirtualThermostat.json` из папки сценария
2. Импортируйте сценарий через интерфейс Sprut.Hub:
   - Сценарии → + → Импортировать шаблон → выберите скачанный файл
3. Вернитесь в настройки виртуального устройства
4. Перейдите в раздел **"Логика"**
5. Выберите сценарий **"🌡️ Виртуальный термостат"**
6. Настройте параметры сценария (см. ниже)
7. Активируйте сценарий переключателем **"Активно"**

### Шаг 6: Настройка параметров

#### Параметр "Датчик температуры"
Выберите датчик температуры, с которого будут браться показания для термостата. Список автоматически формируется из всех устройств типа TemperatureSensor или Thermostat с характеристикой CurrentTemperature.

**Важно:** Значение температуры с датчика будет использоваться как текущая температура термостата.

#### Параметр "Реле нагрева"
Выберите реле (выключатель или розетку) для управления обогревателем. Реле будет включаться, когда термостат перейдёт в режим нагрева.

**Важно:** Если нужен только режим охлаждения, можно оставить пустым.

#### Параметр "Реле охлаждения"
Выберите реле (выключатель или розетку) для управления кондиционером. Реле будет включаться, когда термостат перейдёт в режим охлаждения.

**Важно:** Если нужен только режим нагрева, можно оставить пустым.

#### Параметр "Разница температур для вентилятора"
Шаг разницы температур для изменения скорости вентилятора в градусах Цельсия. 

**Логика работы:**
- При разнице от 0 до установленного шага (например, 0.5°C) — скорость 1 (Тихо)
- От шага (0.5°C) до 2×шага (1°C) — скорость 2 (Медленно)
- От 2×шага (1°C) до 3×шага (1.5°C) — скорость 3 (Средне)
- От 3×шага (1.5°C) до 4×шага (2°C) — скорость 4 (Быстро)
- От 4×шага (2°C) и выше — скорость 5 (Максимум)

**Важно:** Управление вентилятором работает только если термостат поддерживает характеристику Скорость вентилятора (C_FanSpeed).

**Диапазон значений:** от 0.01 до 5.0 градусов, шаг 0.1.

#### Параметр "Ручная фиксация скорости вентилятора"
Если включено, то при ручном изменении скорости вентилятора она перестаёт автоматически меняться. Для повторного включения автоматического режима установите скорость **Авто (0)**.

**По умолчанию:** включено.

#### Параметр "Принудительно устанавливать состояние реле при изменении температуры"
Если включено, состояние реле будет принудительно устанавливаться при каждом изменении температуры, а не только при изменении режима термостата.

**Важно:** Сценарий сам отслеживает, когда реле не в сети, и устанавливает нужное состояние, когда оно подключается к хабу — для этого данную опцию включать не нужно. Используйте её только если базовая логика "Обычный термостат" по какой-то причине не реагирует на изменения температуры так, как ожидается.

**По умолчанию:** выключено.

#### Параметр "Поведение при отказе датчика температуры"
Что делать с реле, если от датчика температуры не поступали данные дольше заданного времени.

**Значения:**
- **Отключать** (по умолчанию) — выключить оба реле (нагрев и охлаждение). Безопасное поведение: при отказе датчика отопление/кондиционер выключаются, чтобы не сжечь дом и не заморозить пустую квартиру.
- **Включать** — включить реле, соответствующее целевому режиму термостата. В режиме нагрева включается реле нагрева, в режиме охлаждения — реле охлаждения. Подходит для критичных систем (например, защита от замерзания), где при потере данных лучше продолжать греть.

**Важно:** После восстановления данных с датчика (любой новый callback от подписки) управление реле возвращается в обычный режим автоматически.

#### Параметр "Время до отказа (минуты)"
Через сколько минут отсутствия данных с датчика температуры считать его отказавшим.

**Диапазон значений:** минимум 15 минут, кратно 15 минутам. По умолчанию 240 (4 часа).

**Важно:** Проверка состояния датчика выполняется по cron каждые 15 минут. Поэтому фактическая задержка применения поведения при отказе — от `failureTimeout` до `failureTimeout + 15` минут.

### Шаг 7: Активация и настройка температуры

1. Вернитесь на экран управления термостатом
2. Установите **Целевую температуру** (желаемую температуру в помещении)
3. Выберите **Целевой режим** (Нагрев, Охлаждение, Автоматически или Выключен)
4. Активируйте термостат

## 🔧 Как это работает

### Логика управления реле

Сценарий управляет реле в зависимости от состояния термостата:

1. **Выключено** (targetState = 0 или currentState = 0):
   - Оба реле выключены

2. **Нагрев** (targetState = 1 или 3, currentState = 1):
   - Реле нагрева включено
   - Реле охлаждения выключено

3. **Охлаждение** (targetState = 2 или 3, currentState = 2):
   - Реле нагрева выключено
   - Реле охлаждения включено

### Получение данных с датчика

Сценарий автоматически:
- Подписывается на изменения датчика температуры
- Обновляет значение CurrentTemperature на термостате при каждом изменении
- Проверяет состояние датчика (в сети ли он)
- Обновляет значение каждый день в полночь (cron задача)
- Каждые 15 минут проверяет, как давно от датчика поступали данные, и при превышении времени до отказа применяет выбранное поведение (см. опции **Поведение при отказе датчика температуры** и **Время до отказа**)

### Автоматическое управление вентилятором

Сценарий автоматически управляет скоростью вентилятора на основе разницы между текущей и целевой температурой:

- **Чем больше разница** — тем выше скорость вентилятора
- **При выключенном термостате** — скорость устанавливается в 1 (минимальная)
- **При ручной установке скорости** (если включена фиксация) — автоматическое управление отключается до установки Авто (0)

### Подписка на состояние реле

Сценарий отслеживает состояние реле (в сети ли они) и автоматически обновляет их состояние при восстановлении связи.

## 💼 Примеры использования

### Пример 1: Простой обогреватель

**Задача:** Автоматически включать обогреватель, когда температура ниже 22°C.

**Настройка:**
1. Создайте виртуальное устройство "Термостат"
2. В режимах отключите "Охлаждение" и "Автоматически"
3. В сценарии выберите:
   - Датчик температуры: ваш датчик
   - Реле нагрева: реле обогревателя
   - Реле охлаждения: оставьте пустым
4. На термостате установите:
   - Целевая температура: 22°C
   - Режим: Нагрев

**Результат:** Обогреватель будет автоматически включаться при температуре ниже 22°C и выключаться при достижении целевой температуры.

### Пример 2: Кондиционер

**Задача:** Автоматически включать кондиционер, когда температура выше 25°C.

**Настройка:**
1. Создайте виртуальное устройство "Термостат"
2. В режимах отключите "Нагрев" и "Автоматически"
3. В сценарии выберите:
   - Датчик температуры: ваш датчик
   - Реле нагрева: оставьте пустым
   - Реле охлаждения: реле кондиционера
4. На термостате установите:
   - Целевая температура: 25°C
   - Режим: Охлаждение

**Результат:** Кондиционер будет автоматически включаться при температуре выше 25°C и выключаться при достижении целевой температуры.

### Пример 3: Полнофункциональный термостат

**Задача:** Поддерживать температуру в диапазоне 20-24°C (нагревать ниже 20°C, охлаждать выше 24°C).

**Настройка:**
1. Создайте виртуальное устройство "Термостат"
2. В режимах включите "Автоматически"
3. В сценарии выберите:
   - Датчик температуры: ваш датчик
   - Реле нагрева: реле обогревателя
   - Реле охлаждения: реле кондиционера
   - Разница температур для вентилятора: 0.5°C (опционально)
4. На термостате установите:
   - Целевая температура: 22°C (середина диапазона)
   - Режим: Автоматически

**Результат:** Система будет автоматически нагревать при температуре ниже целевой и охлаждать при температуре выше целевой, поддерживая комфортную температуру.

### Пример 4: Термостат с автоматическим вентилятором

**Задача:** Термостат с автоматическим управлением скоростью вентилятора в зависимости от разницы температур.

**Настройка:**
1. Настройте термостат как в примере 3
2. Убедитесь, что термостат поддерживает характеристику C_FanSpeed
3. Установите "Разница температур для вентилятора": 0.5°C
4. Включите "Ручная фиксация скорости вентилятора"

**Результат:** 
- При разнице 0-0.5°C — скорость 1 (Тихо)
- При разнице 0.5-1°C — скорость 2 (Медленно)
- При разнице 1-1.5°C — скорость 3 (Средне)
- При разнице 1.5-2°C — скорость 4 (Быстро)
- При разнице 2°C и выше — скорость 5 (Максимум)

При ручной установке скорости автоматическое управление отключается до установки Авто (0).

## ⚠️ Важные замечания

1. **Базовый сценарий:** Перед применением сценария "🌡️ Виртуальный термостат" необходимо активировать базовый сценарий "Обычный термостат" в настройках виртуального устройства.

2. **Режимы работы:** Убедитесь, что в настройках виртуального устройства включены только нужные режимы. Режим "Автоматически" нужен только если используются оба режима (нагрев и охлаждение).

3. **Подписка на датчик:** Сценарий автоматически подписывается на изменения датчика температуры. При первом запуске вы увидите сообщение о подключении датчика.

4. **Мониторинг датчика:** Сценарий отслеживает состояние датчика:
   - Проверяет, в сети ли датчик
   - Каждые 15 минут (cron) проверяет, как давно поступали данные. При превышении времени до отказа применяет выбранное поведение (отключение или включение реле) и пишет ошибку в лог. После восстановления данных управление реле возвращается в обычный режим.

5. **Обновление данных:** Значение температуры обновляется:
   - При каждом изменении показаний датчика
   - Каждый день в полночь (автоматическое обновление)

6. **Типы реле:** В качестве реле можно использовать:
   - Выключатели (Switch)
   - Розетки (Outlet)

7. **Управление вентилятором:** Автоматическое управление скоростью вентилятора работает только если:
   - Термостат поддерживает характеристику C_FanSpeed
   - Не установлена ручная фиксация скорости
   - Скорость не установлена вручную (или установлена в Авто (0))

8. **Подписка на реле:** Сценарий отслеживает состояние реле и автоматически обновляет их при восстановлении связи.

## 🔄 Как это работает

1. **Инициализация:** При запуске сценарий подписывается на изменения датчика температуры, создаёт cron задачу для ежесуточного обновления и cron задачу для проверки отказа датчика (каждые 15 минут).

2. **Получение данных:** При изменении показаний датчика значение автоматически устанавливается на термостат.

3. **Управление реле:** При изменении состояния термостата (нагрев/охлаждение) сценарий включает/выключает соответствующие реле.

4. **Управление вентилятором:** При изменении температуры автоматически вычисляется разница с целевой температурой и устанавливается соответствующая скорость вентилятора.

5. **Мониторинг:** Сценарий отслеживает состояние датчика и реле, предупреждает о проблемах с подключением или отсутствии данных.

6. **Отказ датчика:** Каждые 15 минут сценарий проверяет, как давно поступали данные с датчика. При превышении опции **Время до отказа** применяется выбранное поведение (выключение или включение реле). При следующем callback подписки управление реле возвращается в обычный режим.

## 📝 Примечания

- Сценарий работает только с виртуальным устройством типа "Термостат"
- Требуется активация базового сценария "Обычный термостат"
- Поддерживает работу только с нагревом, только с охлаждением или с обоими режимами
- Автоматически отслеживает состояние датчика и реле, предупреждает о проблемах
- Обновляет значение температуры при каждом изменении и ежедневно в полночь
- Поддерживает автоматическое управление скоростью вентилятора на основе разницы температур
- Поддерживает ручную фиксацию скорости вентилятора с возможностью возврата к автоматическому режиму
